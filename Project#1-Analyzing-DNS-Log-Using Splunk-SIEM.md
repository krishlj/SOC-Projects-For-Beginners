# Analyzing DNS Log Files Using Splunk SIEM

## Introduction

DNS (Domain Name System) logs provide valuable information about domain requests generated by systems within a network. Security analysts can use these logs to identify suspicious domains, unusual DNS traffic, repeated failed requests, malware communication, and other potential security threats.

Splunk SIEM helps security teams collect, search, analyze, and monitor DNS events from a centralized platform.

## Prerequisites

Before analyzing DNS logs in Splunk, ensure the following:

- Splunk Enterprise is installed and configured.
- You can access the Splunk Web interface.
- A sample DNS log file is available.
- You have permission to upload data into Splunk.
- The log file contains details such as source IP, destination IP, domain name, query type, and response code.

## Steps to Upload Sample DNS Log Files to Splunk SIEM

### 1. Prepare the Sample DNS Log File

- Obtain a sample [DNS log file](https://www.secrepo.com/maccdc2012/dns.log.gz) in a supported format such as `.log`, `.txt`, or `.csv`.
- Extract the downloaded `.gz` file if required.
- Confirm that the file contains valid DNS events.
- Check whether the logs include source IP addresses, destination IP addresses, domain names, query types, and response codes.
- Save the file in a location accessible from the system running Splunk.

### 2. Upload the Log File to Splunk

- Log in to the Splunk Web interface.
- Navigate to **Settings** > **Add Data**.
- Select **Upload** as the data input method.

### 3. Choose the DNS Log File

- Click **Select File**.
- Locate and select the prepared DNS log file.
- Click **Next** to continue.

### 4. Set the Source Type

- Open the **Set Source Type** section.
- Review how Splunk displays and separates the events.
- Select an existing DNS source type or create a custom source type.
- For this project, use `dns_sample`.

### 5. Review the Input Settings

- Set the index to `dns_logs`.
- Enter a suitable host name for the system that generated the logs.
- Keep the detected file name as the source.
- Confirm that the source type is `dns_sample`.
- Ensure that all settings match the uploaded DNS log file.

### 6. Submit the Log File

- Click **Review** after completing the configuration.
- Verify the index, host, source, and source type.
- Click **Submit** to upload the file into Splunk.
- Select **Start Searching** after the upload is complete.

### 7. Verify the Upload

- Open the Splunk Search application.
- Set the time range to **All time** because the sample file may contain older events.
- Run the following SPL query:

```spl
index=dns_logs sourcetype=dns_sample
```

If you used a different index or source type, replace `dns_logs` and `dns_sample` with the correct values.

To check the total number of uploaded events, run:

```spl
index=dns_logs sourcetype=dns_sample
| stats count AS total_dns_events
```

## Steps to Analyze DNS Log Files in Splunk SIEM

### 1. Search for DNS Events

- Open the Splunk Search application.
- Enter the following query to display all uploaded DNS events:

```spl
index=dns_logs sourcetype=dns_sample
```

### 2. Extract Relevant DNS Information

- Identify important DNS fields such as source IP, destination IP, domain name, query type, and response code.
- Run the following query to find events containing common DNS-related keywords:

```spl
index=dns_logs sourcetype=dns_sample
| regex _raw="(?i)\b(dns|domain|query|response|port 53)\b"
```

The regular expression searches the raw event data for DNS-related terms.

The exact field names may differ depending on the DNS data source. Common field names include:

- `src_ip` or `src`
- `dest_ip` or `dest`
- `fqdn`, `query`, or `domain`
- `query_type`
- `rcode`

### 3. Identify DNS Activity Spikes

- Look for sudden increases or unusual changes in DNS traffic.
- Run the following query:

```spl
index=dns_logs sourcetype=dns_sample
| timechart span=5m count AS dns_queries
```

This query displays the number of DNS events recorded during each five-minute period.

Unexpected spikes may be caused by malware, automated scripts, DNS scanning, application misconfiguration, or command-and-control communication.

### 4. Find the Top DNS Sources

- Run the following query to identify frequently requested domains and the source systems generating those requests:

```spl
index=dns_logs sourcetype=dns_sample
| top limit=20 fqdn, src_ip
```

- To view only the most frequently requested domains, use:

```spl
index=dns_logs sourcetype=dns_sample
| top limit=10 fqdn
```

- To view the systems generating the most DNS requests, use:

```spl
index=dns_logs sourcetype=dns_sample
| top limit=10 src_ip
```

If your log uses `query` instead of `fqdn`, replace the field name in the search.

### 5. Investigate Suspicious Domains

- Search for domains associated with suspicious or known malicious activity.
- Use an approved threat-intelligence or reputation service such as [VirusTotal](https://www.virustotal.com/) to gather additional information about a domain.
- Run the following query:

```spl
index=dns_logs sourcetype=dns_sample fqdn="suspicious-domain.example"
```

Replace `suspicious-domain.example` with the domain under investigation.

To identify the systems that requested the domain, use:

```spl
index=dns_logs sourcetype=dns_sample fqdn="suspicious-domain.example"
| stats count AS query_count BY src_ip
| sort - query_count
```

A suspicious reputation result should not be treated as final proof of malicious activity. Correlate the DNS event with firewall, endpoint, proxy, and authentication logs before making a final decision.

## Conclusion

Analyzing DNS logs using Splunk SIEM helps security professionals understand network activity and identify potential security threats. By monitoring frequently requested domains, active source systems, DNS traffic spikes, and suspicious domain requests, analysts can detect abnormal behaviour and investigate possible security incidents.

The searches and detection thresholds used in this project can be customized according to the available log format, network size, and normal DNS activity of the environment.

Happy analyzing!
